4. Session & Header Hardening

The portal is protected against Session Hijacking and common Web Vulnerabilities.

  • Session Fingerprinting: Validates browser agent and IP address on every request.
  • Security Headers: Includes HSTS, X-Frame-Options (Clickjacking protection), and X-Content-Type-Options.
  • Cookie Hardening: All session cookies are set to HttpOnly and SameSite: Lax.

5. Administrator Action Guide

To view audit trails or manage security:

  1. Navigate to Security Logs in the admin panel to see document access trails.
  2. Check Audit Logs for failed login attempts and IP blocks.
  3. Use the Self-Service Unlock link in emails to assist users who are legitimately locked out.
